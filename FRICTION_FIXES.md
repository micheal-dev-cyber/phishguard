## P1 â€” HIGH IMPACT

### 1. Suspicious URL patterns miss .xyz and other common phishing TLDs
**File**: `src/detector.py:14-27`
**Issue**: `SUSPICIOUS_URL_PATTERNS` only checks for `.com` TLDs (e.g., `r'secure-.*\.com'`, `r'verify-.*\.com'`). A phishing URL using `.xyz`, `.top`, `.club`, `.site`, `.gq`, `.ml`, `.cf`, `.tk`, `.online` passes through unflagged.
**Impact**: `suspicious_url_count` stays 0, losing up to 25 risk score points from URL analysis. Emails with only a suspicious URL (no keyword triggers) may score lower than they should.
**Fix**: Add patterns for `.xyz`, `.top`, `.click`, `.gq`, `.ml`, `.cf`, `.tk`, `.work`, `.review`, `.download`, `.help`, or use the existing `link_checker.check_url_safety()` which already correctly flags these.

### 2. No upgrade button on pricing plan cards
**File**: `src/auth.py:880-899`
**Issue**: Pricing cards in the landing page are static HTML. Users see plans with prices but no "Choose Plan" or "Subscribe" button per card. They must scroll past FAQ, comparison table, and footer to find the final CTA.
**Impact**: Drop-off between pricing awareness and conversion. Users have to hunt for where to sign up.
**Fix**: Add a "Start Free Trial" or "Choose {plan}" button on each pricing card.

### 3. Demo results missing key value props
**File**: `src/auth.py:575-593` (`_show_demo_results`)
**Issue**: Demo shows only risk score, severity, and keyword hits. VirusTotal cross-reference, OSINT investigation, AI narrative, PDF report, and psychological trigger analysis are all hidden behind signup with no preview.
**Impact**: Users can't see the differentiators that justify paying. The demo proves "it works" but doesn't demonstrate *why PhishGuard is better* than other tools.
**Fix**: Show blurred/summarized previews of gated features with a "Sign up to unlock full analysis" overlay.

## P2 â€” MEDIUM IMPACT

### 4. No plan comparison / feature matrix
**File**: `src/auth.py:867-902`
**Issue**: Plans listed with features inline but no side-by-side feature comparison table. Users can't easily compare what each plan includes.
**Fix**: Add a comparison matrix below the pricing cards showing which features are in which plan.

### 5. Weak passwords accepted at backend
**File**: `src/tenants.py:153` (`create_tenant`)
**Issue**: Frontend enforces 8-char minimum but backend has no password strength validation. A direct API call or bypass could create accounts with `"ab"` as password.
**Fix**: Add backend-side password length check in `create_tenant`.

### 6. Login lockout threshold may be too permissive
**File**: `src/tenants.py` (lockout logic)
**Issue**: In testing, 8 rapid wrong attempts didn't trigger lockout. May require more attempts or have a wider time window.
**Fix**: Verify lockout threshold is appropriate (e.g., 5 attempts in 5 minutes).
